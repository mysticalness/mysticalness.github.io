## 크롬 개발자 도구란?
* 크롬 개발자 도구
  * 크롬에서 제공하는 개발자용 도구
  * 웹페이지에서 일어나는 거의 모든 일을 확인할 수 있는 강력한 개발 도구
  * 웹사이트를 제대로 디버깅 하고 싶다? 시크릿 모드 또는 프라이빗 도구라 불리는 `개인정보 보호 모드`에서 페이지와 개발자 도구를 여는 것을 권장.



## 요소 탭
* 요소 화면
  * 단순히 HTML을 보는 것 뿐만 아니라 직접 코드 수정 가능
  * 요소에 속성을 추가, 수정, 삭제 혹은 해당 요소를 스크린숏으로 캡처하거나 숨기는 등. DOM과 관련된 다양한 작업을 수행할 수 있음.

핫 리로딩(hot reloading) : 처음부터 x 변경사항만 표시하여 적용


핫 리로드

핫 리로드는 앱을 처음부터 다시 시작하지 않고 새로운 코드 변경에 따른 코드 변경 사항 만 표시하며 변경된 코드에만 적용됩니다.



### 요소 정보
* 스타일
* 계산됨
* 레이아웃
* 이벤트 리스너
* DOM 중단점
* 속성
* 접근성



## 소스 탭
* 디버깅 가능
* 소스코드를 오염시키지 않으므로 더 유용
* 소스 코드의 오른쪽
  * 감시
  * 중단점
  * 범위
  * 호출스택
  * 전역 리스너
  * XHR/가져오기
* 스코프, 호출 스택 등 현재 자바스크립트가 실행되고 있는 구조도 확인 가능.


### 네트워크 탭
* HTTP요청, 웹 소켓 등 웹페이지가 외부 데이터와 통신하는 정보를 확인하고 싶다면? 네트워크 탭
* 네트워크 탭을 통해 집중적으로 확인해 봐야 하는 점
   * 불필요한 요청 또는 중복되는 요청이 없는지
   * 웹페이지 구성에 필요한 리소스 크기가 너무 크지 않은지
   * 리소스를 불러오는 속도는 적절한지 또는 너무 속도가 오래 걸리는 리소스는 없는지
   * 리소스가 올바른 우선순위로 다운로드되어 페이지를 자연스럽게 만들어가는지
* 네트워크 탭을 통해 불필요한 네트워크 요청이 없는 지 확인한다면 더욱 빠르고 쾌적한 웹페이지를 만드는데 도움이 된다!


### 메모리 탭
* 애플리케이션에서 발생하는 메모리 누수, 속도 저하, 혹은 웹페이지 프리징 현상을 확인할 수 있는 유용한 도구


* 프리징 현상

프리징(freezing) 또는 응답 없음은 컴퓨터 환경에서 하나의 컴퓨터 프로그램나 시스템 전체가 입력에 대한 응답을 중단할 때 일어난다. 행(hang)이라고도 한다. 가장 흔히 마주치는 상황을 하나 예로 들면 그래픽 사용자 인터페이스를 갖춘 워크스테이션이 있다.


* 프로파일링 작업을 거쳐야 원하는 정보를 볼 수 있다.
* 프로파일 유형 세 가지
     * 힙 스냅샷
     * 타임라인의 할당 계측
     * 할당 샘플링


* 자바스크립트 인스턴트 VM 선택
  * 개발자가 디버깅하고 싶은 자바스크립트 VM환경을 선택

* 힙 스냅샷 
  * 현재 페이지의 메모리 상태를 확인해 볼 수 있는 메모리 프로파일 도구
  * 사진으로 촬용하듯 메모리 현황을 보여 줌.
  
* 타임라인 할당 계측
  * 시간의 흐름에 따라 메모리 변화를 확인할 수 있는 기능.
  * 시간 흐름에 따른 변화를 모두 기록하기 때문에 상대적으로 많은 부담이 발생.
  * 메모리 변화, 메모리 변수, 변수가 어느 정도 크기를 차지하고 있는지 등을 확인 가능.

* 할당 샘플링
  * 타임라인 할당 계측과 비슷 
but
  * 자바스크립트 실행 스택별로 분석 가능.
  * 분석을 함수 단위로 한다. (타임라인 할당 계측과의 차이점)
  * 프로파일링할 때 브러우저에 주는 부담을 최소화 할 수 있음 -> 장시간에 걸쳐 디버깅을 수행할 때 유리.
  * 메모리 누수 확인이 어려워 힙 스냅샷을 촬영하여 비교하기 어려운 경우,
     오랜 기간 메모리 누수가 의심되어 프로파일링을 장기간 수행해야 하는 경우 사용하면 좋음.
